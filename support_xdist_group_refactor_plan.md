 辯論雙方都有合理論點，但在本問題的具體情境下，A 方案更具可行性：                                                                                                                                                                                                                                                                                                       
                                                                                                                                                                                                                                                                                                                                                                        
  • 問題陳述已明確定義需求：hash 模式下相同 xdist_group 必須同 shard                                                                                                                                                                                                                                                                                                    
  • 改動面積最小（≤ 20 行），不引入新 marker API                                                                                                                                                                                                                                                                                                                        
  • xdist_group 本身已是使用者顯式寫在程式碼上的 grouping intent，重用它比新增 shard_group 有更低的採用摩擦

  def _hash_key_for_item(item: pytest.Item) -> str:                                                                                                                                                                                                                                                                                                                     
      """Return hash key: xdist_group name if present, else node ID.                                                                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                                                                                        
      In hash mode, tests sharing an xdist_group are guaranteed to land                                                                                                                                                                                                                                                                                                 
      on the same shard. This respects the grouping intent already                                                                                                                                                                                                                                                                                                      
      declared via pytest-xdist markers.                                                                                                                                                                                                                                                                                                                                
      """                                                                                                                                                                                                                                                                                                                                                               
      marker = item.get_closest_marker("xdist_group")                                                                                                                                                                                                                                                                                                                   
      if marker:                                                                                                                                                                                                                                                                                                                                                        
          # Support both positional and keyword argument forms:                                                                                                                                                                                                                                                                                                         
          # @pytest.mark.xdist_group("name") and                                                                                                                                                                                                                                                                                                                        
          # @pytest.mark.xdist_group(name="name")                                                                                                                                                                                                                                                                                                                       
          group_name = (                                                                                                                                                                                                                                                                                                                                                
              marker.args[0]                                                                                                                                                                                                                                                                                                                                            
              if marker.args                                                                                                                                                                                                                                                                                                                                            
              else marker.kwargs.get("name", "")                                                                                                                                                                                                                                                                                                                        
          )                                                                                                                                                                                                                                                                                                                                                             
          if group_name:                                                                                                                                                                                                                                                                                                                                                
              return f"xdist_group:{group_name}"                                                                                                                                                                                                                                                                                                                        
      return item.nodeid

 潛在風險與因應                                                                                                                                                                                                                                                                                                                                                         
                                                                                                                                                                                                                                                                                                                                                                        
 風險 1：現有使用 xdist_group 純為 worker fixture 隔離的使用者，shard 分布靜默改變                                                                                                                                                                                                                                                                                      
                                                                                                                                                                                                                                                                                                                                                                        
  • 因應：release note 明確標記 behaviour change；提供 group size warning；考慮在 major version 發布此改動                                                                                                                                                                                                                                                              
                                                                                                                                                                                                                                                                                                                                                                        
 風險 2：大型 group 造成單一 shard 過載                                                                                                                                                                                                                                                                                                                                 
                                                                                                                                                                                                                                                                                                                                                                        
  • 因應：group size warning（配套措施 2）；文件說明此 tradeoff；這是 grouping 本身的固有限制，非 A 方案引入的新問題                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                                                                                        
 風險 3：未來需要「只 xdist 分組、不 shard 分組」的解耦需求                                                                                                                                                                                                                                                                                                             
                                                                                                                                                                                                                                                                                                                                                                        
  • 因應：A 方案不鎖死未來擴充。若該需求出現，可引入 shard_group 作為 override（優先級高於 xdist_group），或加入 --shard-ignore-xdist-group CLI flag，屆時有真實需求驅動而非預防性設計                                                                                                                                                                                  
                                                                                                                                                                                                                                                                                                                                                                        
 風險 4：pytest-xdist 未來改變 marker 名稱或參數格式                                                                                                                                                                                                                                                                                                                    
                                                                                                                                                                                                                                                                                                                                                                        
  • 因應：在 _hash_key_for_item 加清楚註解說明依賴關係；marker 讀取邏輯集中在單一函式，未來改動點明確
